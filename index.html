1.Singleton Pattern singleton allows only a single instance of itself to be created and gives access to that created instance.
2.Lazy instantiation in Singleton Pattern Singleton example.

4.Factory PatternDefine an interface for creating an object, but let subclasses decide which class to instantiate.
5.Abstract factory Pattern Provide an interface for creating families of related or dependent objects without specifying their concrete classes.
6.Facade Pattern Provide a unified interface to a set of interfaces in a subsystem. Facade defines a higher-level interface that makes the subsystem easier to use.
7.Proxy Pattern Provide a surrogate or placeholder for another object to control access to it.
8.Observer Pattern Define a one-to-many dependency between objects where a state change in one object results in all its dependents being notified and updated automatically.
9.Observer Pattern in Publisher and Subscriber example Observer example.
10.Command Pattern Encapsulate a request as an object, thereby allowing for the parameterization of clients with different requests, and the queuing or logging of requests. It also allows for the support of undoable operations.
11.Command pattern example. Command pattern example.
12.Command Pattern Example_1 Command pattern example.
13.Template Pattern Define the skeleton of an algorithm in an operation, deferring some steps to subclasses. Template method lets subclasses redefine certain steps of an algorithm without changing the algorithm's structure
14.State Pattern Allow an object to alter its behavior when its internal state changes. The object will appear to change its class.
15.Model View Controller software architectural pattern that divides a given application into three interconnected parts. This is done to separate internal representations of information from the ways information is presented to, and accepted from, the user.The MVC design pattern decouples these major components allowing for efficient code reuse and parallel development.
